Installation

Tout se fait via des prompts — installez, définissez votre projet, lancez les agents

Nouveau Nouveau projet

Pas encore de code — vous partez de zéro

1Créez un repo git et ouvrez Claude Code dessus
2Copiez ce prompt :
"Clone https://github.com/thomasissa-png/Agent-Team (-b claude/extract-project-context-PK8iz) dans /tmp et installe l'équipe Gradient Agents dans ce projet. Nouveau projet."
3Copiez le prompt "Définir mon projet" (section Prompts → Démarrage), remplissez les champs, collez — Claude crée project-context.md automatiquement
4Lancez @orchestrator ou l'agent de votre choix
Résultat :
20 agents dans .claude/agents/ + CLAUDE.md + project-context.md (généré) + docs/ et src/

Existant Projet existant

Du code est déjà en place — rien n'est écrasé

1Ouvrez Claude Code à la racine de votre repo git
2Copiez ce prompt :
"Clone https://github.com/thomasissa-png/Agent-Team (-b claude/extract-project-context-PK8iz) dans /tmp et installe l'équipe Gradient Agents. Projet existant, ne rien écraser."
3Copiez le prompt "Définir mon projet" (section Prompts → Démarrage), remplissez les champs, collez — Claude crée project-context.md automatiquement
4Lancez l'agent dont vous avez besoin (@fullstack, @seo, etc.)
Résultat :
20 agents ajoutés + CLAUDE.md fusionné + project-context.md (généré) — src/, docs/, package.json intacts

Mise à jour Équipe déjà installée

Les agents évoluent — mettez à jour sans perdre vos livrables

1Ouvrez Claude Code sur votre projet existant
2Copiez ce prompt :
"Mets à jour l'équipe Gradient Agents depuis https://github.com/thomasissa-png/Agent-Team (-b claude/extract-project-context-PK8iz). Écrase les agents, fusionne CLAUDE.md, ne touche pas au reste."
3Vérifiez si project-context.md a de nouveaux champs à remplir
Préservé :
project-context.md + docs/ (livrables) + src/ (code) + agents custom

Sessions & Continuité

Changez de session sans perdre le fil — reprenez exactement où vous en étiez

▶ Reprendre une session

Claude Code conserve l'historique de vos conversations. Reprenez à tout moment :

SituationCommande
Session la plus récenteclaude --continue
Session nomméeclaude --resume nom
claude --resumePicker interactif

Bonne pratique : nommez vos sessions au démarrage avec claude -n "ma-feature" pour les retrouver facilement.

📋 CLAUDE.md — votre mémoire

Ce fichier est chargé automatiquement à chaque session. C'est la fondation de la continuité. Gradient Agents l'installe pour vous avec les instructions de l'équipe.

  • Architecture du projet, stack technique
  • Commandes clés (npm run dev, npm test)
  • Conventions de code et décisions
  • Contexte actuel (feature en cours)
/init → génère un CLAUDE.md à partir de votre codebase

🧠 Mémoire automatique

Claude apprend de vos corrections et accumule les patterns dans un fichier mémoire dédié au projet. Pas besoin de re-expliquer vos préférences.

  • Commandes de build découvertes
  • Patterns de code préférés
  • Conventions de nommage
/memory → voir et gérer la mémoire du projet

🛠 Bonnes pratiques multi-sessions

Un projet complet nécessite souvent plusieurs sessions. Voici comment ne rien perdre :

  • project-context.md conserve toutes les décisions entre sessions
  • docs/orchestration-plan.md documente les phases terminées
  • Le tableau "Historique des interventions" trace tout le travail
  • Commencez chaque session par : "Lis project-context.md, où en est-on ?"
