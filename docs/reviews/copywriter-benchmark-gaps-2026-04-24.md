# Benchmark Copy — NanoCorp vs Polsia : gaps framework Gradient Agents
Date : 2026-05-05 | Analyste : @copywriter

---

## 1. Pattern copy décrypté

Les deux sites ciblent un prospect **Solution-Aware** qui rêve d'une entreprise sans frictions opérationnelles. Le pattern commun : **promesse de délégation totale + bénéfice nocturne** ("while you sleep" = métaphore de la liberté absolue, pas d'un gain d'efficacité). Le registre est volontairement provocateur et colloquial — "no babysitting required" rompt délibérément avec le jargon SaaS. Structure macro : tagline audacieuse → preuve comportementale (chiffres réels, cas d'usage) → friction zéro à l'entrée ("No credit card required"). Framework dominant : **AIDA + PAS imbriqués** — l'attention est captée par l'audace, le problème (babysitting, micromanagement) est agité, la solution est posée comme évidence. Niveau de conscience Schwartz : **Solution-Aware** basculant vers **Product-Aware**.

## 2. Techniques qualité — citations sourcées

**T1 — Tagline binaire (promesse + bénéfice temporel)**
- NanoCorp : *"Autonomous Companies Run by AI Making Money While You Sleep"*
- Polsia : *"AI That Runs Your Company While You Sleep"*
- Formule : [Sujet actif non-humain] + [verbe de valeur économique] + [contexte de liberté maximale]
- La tagline est une équation, pas un slogan. Chaque mot porte un poids de conversion.

**T2 — Microcopy anti-friction colloquial**
- NanoCorp : *"No babysitting required"*
- Technique : objection neutralisée en 3 mots, registre langage parlé, humour latent. Remplace 2 paragraphes de réassurance.

**T3 — Impératif tripartite pour onboarder l'imaginaire**
- NanoCorp : *"Define their mission, hire agents, and watch them execute"*
- Structure : 3 verbes → 3 étapes → cinématique mentale. Le prospect SE VOIT utiliser le produit avant de s'inscrire.

**T4 — Preuve sociale chiffrée + comportementale**
- NanoCorp : *"6,684 NanoCorp companies created in 1 month"* — chiffre précis, pas rond, donc crédible.
- Polsia (via Product Hunt) : *"0 to $1M ARR in 1 month"* — résultat utilisateur, pas promesse produit.
- Distinction clé : social proof = résultat réel utilisateur, pas feature list.

**T5 — CTA minimaliste sans friction**
- NanoCorp : *"Get Started"* (primaire) + *"Learn more"* (secondaire)
- Polsia : *"No credit card required"* sous le CTA principal
- Technique : CTA générique assumé — la promesse hero est tellement forte que le CTA n'a pas besoin de porter le bénéfice. Le sous-texte ("no credit card") traite l'objection engagement.

**T6 — Lexique de délégation totale (champ sémantique dominant)**
- NanoCorp : autonomous, execute, mission, hire, making money, no babysitting
- Polsia : plans, codes, markets, 24/7, no employees, co-founder, never sleeps
- Le champ lexical est celui du management d'équipe humaine appliqué à l'IA — créer une dissonance cognitive productive.

**T7 — Hiérarchie titre → sous-titre → preuve → CTA (4 niveaux, 0 filler)**
- Aucun "welcome to" ou "introducing". La hiérarchie est : promesse maximale → démonstration concrète → chiffre → action.
- Chaque niveau réduit une objection sans la nommer.

**T8 — Storytelling comportemental (cas d'usage réel comme preuve)**
- NanoCorp (Twitter/X fondateur) : *"A user prompted a company that audits LinkedIn profiles. The AI agent built the website, set up Stripe payments, and started selling. Fully autonomous."*
- Format : 3 phrases. Acteur → actions → résultat. Zéro adjectif marketing.

**T9 — Positionnement par rupture catégorielle**
- Les deux sites ne se comparent PAS à des concurrents — ils créent une catégorie ("autonomous company") et s'y positionnent seuls.
- Technique Schwartz avancée : déplacer le prospect de "je cherche un outil" vers "je veux une autonomous company".

**T10 — Anti-pattern évité : sur-abstraction fonctionnelle**
- Ce que font bien les deux sites : AUCUNE feature list en above the fold. Aucun "powered by GPT-4", aucun "99.9% uptime", aucun "easy to use".
- L'abstraction reste au niveau du bénéfice de vie, pas de la fonctionnalité.

## 4. Enrichissements proposés

**E1 — Formule tagline hero (section : Frameworks de copywriting)**
Delta : +6 lignes. Ajouter un tableau "Formules de tagline hero" avec 4 patterns testés :
- [Qui fait quoi] + [bénéfice temporel/contextuel]
- [Transformation] + [sans la friction redoutée]
- [Catégorie nouvelle] + [preuve de résultat]
- [Verbe d'action acteur IA] + [liberté maximum]
Règle : la tagline est une équation. Chaque mot porte un poids. Tester A/B obligatoire avant lock.
Bénéfice : sorties taglines mémorables vs outputs actuels "solution complète pour [secteur]" (générique).

**E2 — Microcopy comme outil d'objection (section : UX writing — nouvelle sous-section)**
Delta : +8 lignes. Protocole : identifier les 3 objections principales → pour chacune, rédiger 1 microcopy de 2-5 mots en ton colloquial → placer sous CTA, sur placeholder, ou en caption d'image.
Exemples calibrés : "No card. No commitment. Just go." / "Tes concurrents n'attendent pas" / "5 minutes top."
Règle : le microcopy anti-friction remplace la réassurance longue. 1 phrase colloquiale > 1 paragraphe formel.
Bénéfice : neutraliser les micro-hésitations sans alourdir la page. Technique invisible, impact maximal.

**E3 — Technique de visualisation d'usage (section : Frameworks — nouveau pattern)**
Delta : +5 lignes. Ajouter le pattern "Cinématique mentale" : 3 impératifs chronologiques qui projettent le prospect dans l'usage. Structure : [Action d'initialisation] → [Action de délégation] → [Observation du résultat].
Règle : les 3 verbes doivent être distincts, progressifs, et évocateurs. Le prospect doit SE VOIR utiliser le produit.
Bénéfice : conversion above the fold sans démo. Le cerveau du prospect teste le produit avant de cliquer.

**E4 — Protocole category creation (nouvelle section : Positionnement catégoriel)**
Delta : +10 lignes. Avant calibration sectorielle, poser la question : "Pouvons-nous NOMMER une catégorie nouvelle ?" Si oui : définir le nom de catégorie, ses 3 attributs distinctifs, et le copy qui installe cette catégorie dans l'esprit du prospect.
Protocole : 1) nommer la catégorie (2-3 mots) → 2) définir ce qui la distingue des catégories existantes → 3) produire le hero copy qui installe la catégorie avant de vendre le produit.
Bénéfice : si réussi, l'entreprise est seule dans sa catégorie. Le concurrent le plus dangereux n'existe pas encore.

**E5 — Règle "bénéfice de vie > feature" (section : Standard de livraison, nouvelle checkbox)**
Delta : +3 lignes. Ajouter au checklist auto-évaluation : "Le above the fold parle-t-il d'un bénéfice de VIE (liberté, argent, temps, statut) plutôt que d'une feature produit ? Si une feature apparaît en H1/H2 above the fold → reprendre."
Exemples : "Making money while you sleep" (bénéfice de vie) vs "Automated Stripe integration" (feature).
Bénéfice : forcer la sortie du registre feature-list qui convertit mal sur audiences froides (Unaware/Problem-Aware).
